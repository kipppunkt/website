Software engineering is undergoing a paradigm shift. AI agents have already moved coding out of the IDE — engineers spend most of their time prompting agents in terminals. But that's just an intermediary step.

## The shift

If agents are good enough to write code by themselves, the human part of engineering shifts elsewhere: **refining ideas into specs** and **reviewing outcomes**. A new engineering role emerges — someone who is product manager, architect, and engineer at once, steering AI agents instead of writing code by hand.

A single high-performing engineer with the right toolchain can ship what used to require a team.

## The full loop

kipp•punkt covers the full AI-assisted engineering loop: from fuzzy idea to shipped code.

```
idea → refine → requirement → build → outcome → review → merge
```

The human operates at the edges — shaping intent and approving results. Everything in between is automated.

### Refine — turn ideas into requirements

Tag kipp•punkt on a GitHub issue and it starts a refinement conversation: clarifying questions, edge cases, proposed requirements. Once you approve, the requirement is handed off for implementation.

Looking ahead, refinement will extend beyond text. Voice-based sessions — joining a call on Teams, Zoom, or whatever you already use — will let you talk through ideas directly from a GitHub issue. No new app. Existing infrastructure, new capability.

### Build — autonomous implementation

Takes requirements, implements them, creates PRs, and incorporates feedback from code review. Runs around the clock. This is the most mature piece of the toolchain today.

### Review — close the feedback loop

The least defined piece, and intentionally so. The conviction: the way we review AI-generated code will change fundamentally. What that looks like is still emerging.

Possibilities range from a review assistant that helps spot issues in PRs, to a long-term memory system that learns from human feedback over time and incorporates it *before* the PR is even created. This area stays exploratory until the real need crystallizes.

## Integration philosophy

**All the right tools already exist.** We know how to write requirements, display diffs, review code, and hold refinement meetings. AI doesn't change any of that. Nothing needs to be reinvented.

kipp•punkt integrates with the tools you already use — GitHub, your terminal, your video call platform — as smoothly as possible. No new UIs unless truly necessary. Features first, chrome later.

## Who it's for

High-performing engineers, founders, and indie devs. People who can think across product, architecture, and implementation. kipp•punkt amplifies that kind of person.

Small teams are a natural extension, but individual engineers are the initial focus.
